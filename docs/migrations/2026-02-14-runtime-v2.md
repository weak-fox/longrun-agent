# Runtime v2 Migration Notes (Phase 2)

Date: 2026-02-14

## Summary

This migration finalizes the multi-backend architecture and removes legacy article orchestration entrypoints.

## Breaking Changes

- Removed `article-run` CLI command.
- Removed legacy modules:
  - `src/longrun_agent/article/runner.py`
  - `src/longrun_agent/article/client.py`
- Unified all session execution through `Harness` (`run-session` / `run-loop`).

## New Runtime Model

- Backend selection:
  - `codex_cli`
  - `claude_sdk`
- Prompt profile selection:
  - `default`
  - `article`
- Runtime overrides available on CLI:
  - `--backend`
  - `--profile`
  - `--backend-model`

## New Hard-Gate + Remediation Flow

- Added deterministic gate checks:
  - initializer required artifacts
  - coding `feature_list.json` invariants
  - verification command gate
  - optional git clean / commit / progress update gates
- Added remediation artifacts:
  - `.longrun/remediation/session-XXXX.json`
- Added rollback action for invalid coding-phase `feature_list` mutations.

## Config Changes

`longrun-agent.toml` now supports:

- `[runtime]`
- `[backends.codex_cli]`
- `[backends.claude_sdk]`
- `[gates]`

## Migration Actions

1. Replace any `article-run` usage with:
   - `longrun-agent run-loop --backend claude_sdk --profile article ...`
2. Update config to use `[runtime]` and `[backends.*]`.
3. If needed, enable stricter gates via `[gates]`.
